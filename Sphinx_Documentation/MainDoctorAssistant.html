MAIN DOCTOR ASSISTANT CODE DOCS¶

class Doctor_Assistant_1.src.MainAssistantPi.doctorAssistant¶

This class is the doctor Assistant class. It operates the google assistant for the doctor. Here the doctor can ask to see next patient, get just the info of the next patient, and dictate to the pi.

dicatationService()¶

The dictation service. Here the assistant is taking note of the speech, and re-listening. doctor triggers next line by repeating “OK GOOGLE”

done()¶

Here the doctor can now remove the last seen patient from the waiting list. Appointment complete.

main()¶

Here hotwords are detected and sent through as events to process events.

nextInfo()¶

Here the doctor can obtain a breif summary of the next appointment without calling the petient in. Like so, they can check how they are going for time, can they take breaks etc…

nextPatient()¶

Using this function the doctor is able to be told about the next appointment and have the patient called in. NOTE: called in patient is kept as current patient.

nextQuick()¶

This is a faster way to call in the patient. If the doctor wants to save time, and doesn’t want to assistant taking too long to call in the patient. NOTE: called in patient is kept as current patient.

process_event(assistant, event)¶

Here the assistant processes incoming events. Ie: When a hotword is detected.

sendDicatation()¶

Here the dictated speech is sent to the main API for adding the note to the patient record.
